# 明天待辦事項（Hookvox 上線收尾）

## 一、Push 後要不要重新部署？

**不用手動重新部署。**  
Push 到 GitHub 的 `main` 後，Vercel 會**自動**觸發一次新的建置與部署，幾分鐘內就會生效。  
只有自動部署沒跑、或失敗時，才需要到 Vercel 專案裡手動點「Redeploy」。

---

## 二、明天要做的事（依順序）

### 1. 確認程式已 Push
- [ ] 若今天還沒 push 首頁價格修改，先執行：
  ```powershell
  cd "c:\Users\USER\OneDrive\Desktop\viralscript-ai"
  git add src/app/page.tsx
  git commit -m "fix: 首頁價格改為與方案頁一致（699/1599）"
  git push origin main
  ```
- [ ] 到 Vercel 的 Deployments 看是否有新的部署、狀態是否成功

---

### 2. Cloudflare Turnstile 網域（登入/註冊真人驗證）
- [ ] 登入 **Cloudflare Dashboard** → 左側選 **Turnstile**
- [ ] 點你使用的 **Widget**（有 Site Key 的那個）
- [ ] 進入 **Settings** → **Hostname Management** → **Add Hostnames**
- [ ] 新增你目前使用的網址，例如：
  - `hookvox-1yib.vercel.app`
  - 若有自訂網域也一併加入，如 `hookvox.com`
- [ ] 儲存後等 1～2 分鐘，再重新整理登入/註冊頁，真人驗證應可正常載入與通過

（若沒加網域，會出現「真人驗證失敗」或「無法連線至網站」，註冊/登入按鈕會無法送出。）

---

### 3. Vercel 環境變數（正式網址）
- [ ] 打開 Vercel 專案 **hookvox** → **Settings** → **Environment Variables**
- [ ] 新增或修改以下變數（Value 改成你的正式網址，例如 `https://hookvox-1yib.vercel.app`）：
  - `NEXT_PUBLIC_APP_URL` = 你的正式網址
  - `NEXTAUTH_URL` = 你的正式網址
  - `ECPAY_NOTIFY_URL` = 你的正式網址 + `/api/ecpay/notify`
  - `ECPAY_RETURN_URL` = 你的正式網址 + `/payment/success`
- [ ] 確認其他變數也有填（Supabase、DATABASE_URL、綠界 MERCHANT_ID / HASH_KEY / HASH_IV、Anthropic/OpenAI 等）
- [ ] 儲存後到 **Deployments** → 最新一筆右側 **⋯** → **Redeploy**（環境變數改動要重新部署才會生效）

---

### 4. 綠界後台（正式環境）
- [ ] 登入綠界 **廠商後台**
- [ ] 若要用**正式**金流：把「付款完成回傳網址」改成正式環境的 notify 網址  
  例：`https://你的網域.vercel.app/api/ecpay/notify`
- [ ] 確認「返回商店網址」或相關設定是指向你的正式首頁或方案頁
- [ ] （若還在用測試環境）確認測試環境的回傳網址是 ngrok 或目前使用的網址，避免混用

---

### 5. 測試付款流程（建議小額或測試單）
- [ ] 用正式網址打開網站 → 登入/註冊 → 到方案頁
- [ ] 選一個方案、走一次綠界付款（可用測試卡或小額）
- [ ] 確認：付款完成後會跳回 `/payment/success`、方案/額度有更新
- [ ] 若有問題：檢查 Vercel 的 **Functions** 或 **Logs** 看 `/api/ecpay/notify` 是否有被呼叫、是否有錯誤

---

### 6. 可選（有空再做）
- [ ] 在 Vercel 綁定自己的網域（Settings → Domains）
- [ ] 檢查首頁、方案頁、帳單頁的價格與文案是否都正確
- [ ] 寄一封測試信確認「訂閱電子報」或聯絡信箱 (hookvox.support@gmail.com) 正常

---

## 三、快速檢查清單

- 程式已 push，Vercel 自動部署成功  
- **Turnstile 已加入目前網域**（登入/註冊才能驗證）  
- Vercel 環境變數已設好並 Redeploy 一次  
- 綠界回傳網址是正式環境  
- 實際走過一次付款流程且成功  

以上都完成後，就可以當作正式上線收尾完成。
