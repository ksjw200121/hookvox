# 照做步驟：檢查與補強（不用自己判斷，照做就好）

以下都是「你只要照做」的步驟，做完就打勾。

---

## 0. 先安裝新套件（約 1 分鐘）

在專案資料夾執行一次（會套用 Next.js 安全更新 14.2.35 與 glob 覆寫）：

```bash
npm install
```

打完勾：□ 已執行 `npm install`。

---

## 1. 依賴安全檢查（約 1 分鐘）

在專案資料夾打開終端機（PowerShell 或 CMD），執行：

```bash
npm audit
```

- **若顯示 0 vulnerabilities**：不用做任何事，打勾即可。
- **若有 critical 或 high**：先執行  
  `npm audit fix`  
  再跑一次 `npm audit`。若**只剩 1 個 high 且是 `next`**：不用再處理，見下方說明。若是其他套件或 critical，再考慮  
  `npm audit fix --force`  
  （本專案請勿為了解掉 next 那 1 個而跑 `--force`，會升到 Next 16 易壞。）

**專案已做**：Next.js 鎖在 **14.2.35**（Next.js 官方 2025-12-11 安全公告的 14.x 修補版，已修 DoS / 原始碼洩漏）；`glob`、`rollup` 已用 overrides 指定安全版本。

**若 npm audit 仍顯示 1 high（next）**：這是 npm 公告尚未把 14.2.35 標為已修補所致，實際我們已是官方認定的修補版，**可忽略這 1 個**。請勿跑 `npm audit fix --force`（會強制升到 Next 16 並可能導致專案壞掉）。

打完勾：□ 已執行 `npm audit`，必要時已跑 `npm audit fix`（或 `--force`）。

---

## 2. Supabase 資料庫備份（可選）

**去哪裡找：**

1. 登入 [Supabase](https://supabase.com/dashboard)，選你的專案（例如 ksjw200121's Project）。
2. 左側選單往下捲，在 **Database** 區塊底下（或 **Platform** 區塊）找到 **Backups**。
3. 點 **Backups**，就會進入「Database Backups」頁（有 Scheduled backups / Point in time / Restore to new project 那頁）。

**到了之後：**

- **免費方案**：那一頁會寫「Free Plan does not include project backups」，代表免費沒有排程備份，**不用再點其他東西**，到這裡就算完成「找到備份設定」。
- **若需要備份**：可升級 Pro 在同一頁開每日備份，或維持免費、自己用 Table Editor 定期匯出重要表成 CSV。

打完勾：□ 已到 Backups 頁確認過；□ 已知免費方案無內建備份（若有需要再決定是否升級或自行備份）。

---

## 3. 已幫你加好的東西（不用再設定）

這些我已經做在專案裡了，你只要部署就會生效：

- **Vercel Analytics**  
  已加在 `layout.tsx`，部署到 Vercel 後會自動送流量數據到 Vercel 後台，你可在 Vercel 專案裡看「Analytics」分頁。
- **Sentry 錯誤追蹤（選用）**  
  程式已接好，但**預設是關閉**的。之後若你想看錯誤報告：
  1. 到 [sentry.io](https://sentry.io) 註冊並建立一個專案（選 Next.js）。
  2. 複製專案給你的 **DSN**（一串網址）。
  3. 在 Vercel 專案 **Settings → Environment Variables** 新增：
     - 名稱：`NEXT_PUBLIC_SENTRY_DSN`
     - 值：貼上 DSN
  4. 重新部署。之後 Sentry 就會開始收錯誤。

現在不設定 Sentry 也沒關係，網站一樣正常運作。

打完勾：□ 已知 Vercel Analytics 已啟用；□ 已知 Sentry 可選填 DSN 後啟用。

## 4. 綠界審核通過後你要做的事

1. 到綠界後台把 **付款完成回傳網址** 設成：  
   `https://hookvox-1yib.vercel.app/api/ecpay/notify`
2. 自己刷一筆小額測試，確認：
   - 訂單狀態會變成「已付款」、
   - 方案/額度有正確升級、
   - （可選）到 Supabase 的 `payment_webhook_events` 表看得到一筆 SUCCESS。

打完勾：□ 回傳網址已設；□ 已跑過一筆測試付款並確認正常。
