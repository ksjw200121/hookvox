# Hookvox 系統完整度與定價評估

> 供產品決策與迭代參考，可依優先級逐步實作。

---

## 一、系統完整度與流暢度

### 已具備的完整流程 ✅

| 階段 | 狀態 | 說明 |
|------|------|------|
| 登入／註冊／忘記密碼 | ✅ | Supabase Auth、真人驗證、錯誤訊息 |
| 爆款分析 | ✅ | 上傳→開始爆款分析→確認→轉錄+分析、扣分析額度 |
| 腳本／標題／分鏡生成 | ✅ | 依分析結果生成、扣生成額度、3 版本 A/B/C |
| 爆款資料庫 | ✅ | 列表、搜尋、收藏、延伸角度、延伸腳本（依方案限制） |
| 方案與金流 | ✅ | 四方案、綠界、差額升級、週期不重算、次數延續 |
| 帳單 | ✅ | 目前方案、本期額度、到期日、付款記錄、計費說明 |

### 流暢度小結

- **主流程**：分析 → 生成 → 資料庫，路徑清楚；分析頁「上傳後不自動轉錄」已避免誤扣。
- **會員與金流**：升級差額、次數不重置、到期重置、續訂全額，邏輯一致。
- **權限同步**：舊內容依目前方案判斷、FLAGSHIP 延伸腳本已修正，無明顯斷點。

### 可再加強的流暢點（非必須）

- 新用戶第一次進「爆款分析」或「爆款資料庫」時，可加 1～2 句引導（例如：先分析一支影片，再生成腳本）。
- 分析／生成失敗時，錯誤訊息可再區分「額度不足」「網路錯誤」「內容不符」並對應不同引導（如升級、重試、改內容）。

## 二、最建議加的功能（按優先級）

### 高優先（有助「做到完美」、轉換與留存）

1. **方案頁：Creator vs Pro vs 旗艦 功能對照表**
   - **現狀**：方案卡只有條列 feature，Creator 與 Pro 的「實際差異」不夠一眼看懂。
   - **建議**：加一個簡表，例如：
     - 分析+生成次數：50 / 200 / 500
     - 每支影片可生成「延伸腳本」數：1 / 3 / 3
   - **效益**：付費理由更清楚，減少「Pro 只是比較多次？」的疑惑。

2. **Creator 與 Pro 的「功能差」再明顯一點**
   - **現狀**：唯一功能差是「延伸腳本：每支影片 1 個 vs 3 個」。
   - **建議**（擇一或擇二即可）：
     - 方案頁文案寫清楚：「Creator 每支影片可生成 1 個延伸腳本；Pro/旗艦 可生成 3 個。」
     - 或為 Pro 加一個「輕量獨有」：例如「一鍵匯出本週腳本（文字檔）」或「標題一鍵複製全部」，讓 Pro 不只「次數多」還有「效率工具」。
   - **效益**：Pro 的升級動機更明確，定價更好支撐。

3. **控制台／帳單：本期使用趨勢（可選）**
   - **現狀**：有「已用 / 額度」與「本期到期日」。
   - **建議**：若有餘力，可加「本週／本期 分析幾次、生成幾次」的簡單數字或 mini 趨勢（不一定要圖表），讓使用者有感「有在用到額度」。
   - **效益**：提升「有在用」的體感，有利續訂。

### 中優先（加分、不影響主流程）

4. **首次使用引導（onboarding）**
   - 新用戶第一次進入分析頁或資料庫時，短文案或 1 步 tooltip：「上傳影片 → 開始爆款分析 → 生成腳本與標題」。
   - 降低「不知道從哪開始」的流失。

5. **錯誤與空狀態統一**
   - 額度用完、未登入、分析失敗、資料庫無資料等，統一 tone（例如：簡短說明 + 下一步按鈕：升級／登入／重試／去分析）。

### 低優先（有資源再做）

6. **匯出／批次複製**
   - 例如：爆款資料庫某支影片的「所有標題」或「腳本+標題」一次複製或匯出 txt。Pro/旗艦可列為獨有，強化方案差異。

7. **簡單數據回饋（不影響現有邏輯）**
   - 例如：本週「分析 X 支、生成 Y 次」，僅顯示數字即可，不做複雜報表。

## 三、價位與功能是否符合、合理嗎？

### 目前定價與單次成本

| 方案 | 月費 | 分析+生成總額度 | 約單次成本（以總額度計） |
|------|------|------------------|---------------------------|
| FREE | 0 | 6（3+3） | — |
| Creator | 699 | 100（50+50） | 約 7 元/次 |
| 專業版 Pro | 1,599 | 400（200+200） | 約 4 元/次 |
| 旗艦版 | 2,999 | 1,000（500+500） | 約 3 元/次 |

### 與市面對照（主觀參考）

- 純「AI 寫文案／腳本」工具：月費約 500～2,000 很常見。
- 你提供的是：**分析爆款結構 + 腳本 + 標題 + 分鏡 + 延伸角度／腳本 + 資料庫**，屬於「分析+生成+管理」一條龍，功能面高於單純寫稿工具。
- **結論**：在「功能完整、流程可閉環」的前提下，**699 / 1599 / 2999 的定價與功能是匹配的**，可視為合理；若要「做到完美」，重點會是：讓**方案差異**與**價值感**在 UI 與文案上更清楚，而不是再壓低價錢。

### 建議對外說法（一句話版）

- **Creator 699**：適合剛開始做短影音、每月約 50 次分析+生成就夠用的創作者，且有「每支影片 1 個延伸腳本」。
- **Pro 1599**：適合每週產片多、需要「每支影片 3 個延伸腳本」、或想壓低單次成本的重度創作者。
- **旗艦 2999**：適合團隊／代操／多帳號，或每月 500 次仍不夠的極重度使用。

## 四、Creator 和 Pro 功能有差別嗎？夠明顯嗎？

### 現有差異（程式與邏輯）

| 項目 | FREE | Creator | Pro | 旗艦 |
|------|------|---------|-----|------|
| 分析次數/週期 | 3 | 50 | 200 | 500 |
| 生成次數/週期 | 3 | 50 | 200 | 500 |
| 每支影片可生成「延伸腳本」數 | 0 | **1** | **3** | **3** |
| 其餘功能 | 同左 | 腳本+標題+分鏡、資料庫、套用替換 | 同上 | 同上 |

也就是說：**Creator 和 Pro 有差別**，差在「次數」與「每支影片延伸腳本數量」（1 vs 3）；其餘功能一致。

### 是否夠明顯？

- **在程式與權限上**：有差，且已正確實作（爆款資料庫、API 都有依方案限制延伸腳本數）。
- **在方案頁與日常使用上**：多數使用者不會立刻發現「延伸腳本 1 vs 3」的差異，容易覺得 Pro 只是「買更多次數」。

### 建議（讓差別更符合定價）

1. **方案頁與說明文案**
   - 在 Creator / Pro 的 feature 裡**明確寫**：
     - Creator：「每支影片可生成 **1** 個延伸腳本（從 3 個角度選 1 個寫成腳本）」
     - Pro：「每支影片可生成 **3** 個延伸腳本（3 個角度都可寫成腳本，方便 A/B 測試）」
   - 若有「優先客服」或「升級差額」等，保留在 Pro 說明即可。

2. **產品內輕量提示（可選）**
   - 在爆款資料庫「延伸腳本」區，Creator 用到 1 個上限時，提示：「你已用完本支影片的 1 個延伸腳本額度；升級 Pro 可每支影片生成 3 個。」

3. **若想再拉開 Pro 價值**
   - 在不改核心邏輯的前提下，可為 Pro（或 Pro+旗艦）加一個「小功能」，例如：
     - 一鍵匯出本支影片的標題/腳本為文字檔，或
     - 「複製全部 8 個標題」一次帶走。
   - 這樣 Pro 不只有「次數多、延伸腳本多」，還有「效率工具」，價位更好支撐。

---

## 五、總結：要做到完美，優先做什麼？

1. **方案頁**：加「功能對照表」、把「延伸腳本 1 vs 3」寫清楚，讓 Creator vs Pro 差異一眼看懂。
2. **定價**：維持現有 699/1599/2999 可接受，重點是**把現有差異講清楚**，必要時為 Pro 加一個小獨有功能。
3. **流暢度**：主流程與金流已完整；可再補「新用戶引導」與「錯誤/空狀態」統一，屬加分項。
4. **進階**：若要做更完美，再考慮「本期使用趨勢」與「匯出/批次複製」等，不影響現有邏輯。

整體而言，**系統完整度與流暢度已達可商用水準**；「做到完美」的重心建議放在：**方案差異可見、定價理由清楚、小處體驗統一**，而不是再加一大堆新功能。
