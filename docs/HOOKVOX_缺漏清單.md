# Hookvox 還缺少什麼 — 缺漏清單

> 依產品、轉換、信任、法規、成長等面向整理，供排優先級用。

---

## 一、產品與轉換（直接影響「買不買」「會不會續訂」）

### 1. 方案頁：Creator / Pro / 旗艦「功能對照表」還沒有
- **現狀**：方案卡只有條列 feature，Creator 和 Pro 的「次數、延伸腳本數」差異要自己算。
- **建議**：在方案頁加一個簡表，例如：
  - 分析+生成次數：3+3 / 50+50 / 200+200 / 500+500
  - 每支影片延伸腳本數：0 / 1 / 3 / 3
  - 一鍵複製全部標題、匯出 .txt：僅 Pro / 旗艦
- **效益**：付費理由一眼看懂，減少「Pro 只是比較貴？」的疑惑。

### 2. 首頁 / 對外頁沒有連到「服務條款」與「隱私權政策」
- **現狀**：Dashboard 內 footer 有連到 `/terms`、`/privacy`，但**首頁、成功案例、功能教學**的 footer 沒有。
- **建議**：所有對外頁 footer 都加上「服務條款」「隱私權政策」連結（你已有 terms / privacy 頁，只差連過去）。
- **效益**：合規、信任；部分付費管道也會要求官網有這些連結。

### 3. 新用戶第一次進分析頁 / 資料庫：沒有引導
- **現狀**：登入後直接進控制台，若沒看教學可能不知道「先分析再生成」。
- **建議**：第一次進「爆款分析」或「爆款資料庫」時，顯示一次性的短引導（例如 1 句＋「知道了」按鈕），或一個小 tooltip：「上傳影片 → 開始爆款分析 → 生成腳本與標題」。
- **效益**：降低「不知道從哪開始」的流失。

### 4. 額度用完 / 分析或生成失敗時的引導不統一
- **現狀**：各頁對「額度不足」「網路錯誤」的處理方式不完全一致，有的只顯示錯誤訊息，沒有明確下一步。
- **建議**：統一「錯誤／空狀態」的 tone：簡短說明 + 一個明確的下一步按鈕（額度不足 → 升級方案；未登入 → 登入；分析失敗 → 重試或改內容）。
- **效益**：減少卡關感，順便導向升級或重試。

### 5. 付款成功頁較陽春
- **現狀**：付款成功後只顯示「付款成功」+ 幾秒後跳回方案頁。
- **建議**：可加「本期額度已更新」「下次到期日」或「前往爆款分析」的 CTA，讓付費後有明確下一步。
- **效益**：付費完立刻有「可以開始用了」的滿足感。

## 二、信任與法規（讓用戶敢付錢、你合規）

### 6. 首頁 / 註冊頁沒有「服務條款」「隱私權政策」連結
- **同上 2**：對外頁 footer 補上 terms / privacy 連結；註冊流程若有「同意條款」勾選，應可連到條款內文。

### 7. 退款政策沒有獨立說明
- **現狀**：首頁定價區有「數位內容一經提供不適用鑑賞期退款」一句，但沒有「什麼情況可退、如何申請」的集中說明。
- **建議**：在服務條款或獨立「退款政策」頁寫清楚：依消保法不適用鑑賞期、例外處理方式、聯絡方式。付費前或方案頁可放連結。
- **效益**：減少爭議、符合常見電商做法。

### 8. 聯絡方式與客服管道
- **現狀**：有 mailto:hookvox.support@gmail.com 與「聯繫我們／聯繫客服」。
- **建議**：若未來有表單或客服系統，可集中寫在「聯絡我們」或 Help 頁；目前維持 email 也可，但建議在條款／隱私頁都寫明同一聯絡方式。

## 三、內容與溝通（讓價值被看見）

### 9. 方案差異文案不夠具體
- **現狀**：Creator 與 Pro 的「延伸腳本 1 vs 3」在方案卡上沒有直接寫出來。
- **建議**：在方案卡或對照表寫明：「Creator：每支影片可生成 **1** 個延伸腳本」「Pro / 旗艦：每支影片可生成 **3** 個延伸腳本」，必要時加一句「方便 A/B 測試」。
- **效益**：Pro 的價差有明確理由。

### 10. 免費試用「3 次」的說明
- **現狀**：多處寫「免費試用 3 次」或「3 次分析 + 3 次生成」，已算清楚。
- **可選**：在註冊成功或控制台第一次出現時，再強調一次「你現在有 3 次分析、3 次生成，建議先分析一支爆款再生成」，強化路徑。

### 11. FAQ 或常見問題
- **現狀**：沒有集中 FAQ（例如：如何取消訂閱？額度什麼時候重置？升級差額怎麼算？）。
- **建議**：在方案頁下方或獨立「常見問題」區塊／頁面，放 5～10 題，減少客服負擔、也利於轉換。

## 四、體驗與穩定性（小但影響體感）

### 12. 分析／生成中的進度或狀態
- **現狀**：有 loading，但長時間等待時用戶可能不確定是「卡住」還是「還在跑」。
- **建議**：若技術可行，可考慮「轉錄中…」「分析中…」分階段提示，或至少寫「約需 1～2 分鐘，請稍候」。
- **效益**：減少誤以為當掉而關掉頁面。

### 13. 手機版體驗
- **現狀**：RWD 已有，但未全面檢查每個流程在手機上的可操作性。
- **建議**：關鍵流程（註冊、分析、生成、方案頁、付款）在手機上走一遍，確認按鈕、表單、彈窗都好點、好填。
- **效益**：短影音創作者很多用手機，手機體驗好才不會流失。

### 14. 登出或 session 過期後的處理
- **現狀**：有登出與登入狀態；若 token 過期，部分 API 會回 401。
- **建議**：前端對 401 統一處理（例如導向登入頁並帶 redirect），避免用戶看到空白或技術錯誤訊息。
- **效益**：體驗穩定、不嚇人。

## 五、成長與營運（可排較後）

### 15. 推薦／邀請機制
- **現狀**：無。
- **建議**：若有餘力，可做「邀請朋友註冊，雙方各得 N 次額度」或類似方案，利於自然擴散。
- **優先級**：低，可等主流程與轉換都穩再考慮。

### 16. 電子報或等待名單
- **現狀**：無。
- **建議**：若想做行銷或新功能預告，可在首頁或註冊後加「訂閱電子報」或「搶先體驗新功能」留 email。
- **優先級**：低。

### 17. 數據與營運
- **現狀**：有 admin 成本頁、usage 與帳單。
- **建議**：若尚未做，可考慮簡單的「每週註冊數、轉換率、活躍用戶」等 dashboard，方便你決策定價與功能優先級。
- **優先級**：依你營運需求。
